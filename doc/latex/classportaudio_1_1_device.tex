\doxysubsection{Detailed Description}
Class which represents a Port\+Audio device in the \mbox{\hyperlink{classportaudio_1_1_system}{System}}. 

A single physical device in the system may have multiple Port\+Audio \mbox{\hyperlink{classportaudio_1_1_device}{Device}} representations using different \mbox{\hyperlink{classportaudio_1_1_host_api}{Host\+Api}} \textquotesingle{}s though. A \mbox{\hyperlink{classportaudio_1_1_device}{Device}} can be half-\/duplex or full-\/duplex. A half-\/duplex \mbox{\hyperlink{classportaudio_1_1_device}{Device}} can be used to create a half-\/duplex \mbox{\hyperlink{classportaudio_1_1_stream}{Stream}}. A full-\/duplex \mbox{\hyperlink{classportaudio_1_1_device}{Device}} can be used to create a full-\/duplex \mbox{\hyperlink{classportaudio_1_1_stream}{Stream}}. If supported by the \mbox{\hyperlink{classportaudio_1_1_host_api}{Host\+Api}}, two half-\/duplex Devices can even be used to create a full-\/duplex \mbox{\hyperlink{classportaudio_1_1_stream}{Stream}}.

Note that \mbox{\hyperlink{classportaudio_1_1_device}{Device}} objects are very light-\/weight and can be passed around by-\/value.
